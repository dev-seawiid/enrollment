# Project Checklist — 수강신청 서비스

`requirements.md`를 바탕으로 도출된 반드시 구현되어야 하는 항목들입니다.

## 🛠 기술 및 환경 (Technical Requirements)

- [ ] **Next.js App Router** 사용
- [ ] **TypeScript** 기반 개발
- [ ] **SSR (Server Side Rendering)** 구현
- [ ] **Semantic HTML** 준수 및 구조화된 마크업
- [ ] **Flexbox 또는 Grid Layout**을 활용한 모바일 웹 최적화 (max-w-md 등)
- [ ] **테스트 코드** 작성 (로직 및 컴포넌트)
- [ ] **README.md** 작성 (프로젝트 설명, 실행 방법, 기능 목록 등)

---

## 🔐 1. 계정 및 인증 (Account & Auth)

- [ ] **회원 가입 (Sign Up)**
  - [ ] 이름, 이메일, 휴대폰 번호 입력 필드 구현
  - [ ] 휴대폰 번호 포맷팅 및 유효성 검사 (010-0000-0000 등)
  - [ ] 비밀번호 복합 규칙 적용 (6~10자, 영문 소문자, 대문자, 숫자 중 2종 이상 조합)
  - [ ] 회원 유형 (수강생 / 강사) 선택
  - [ ] 가입 완료 후 **강의 목록 페이지로 리다이렉트**
- [ ] **로그인 및 세션 관리**
  - [ ] 로그인 기능 구현 및 인증 토큰/세션 유지
  - [ ] 비로그인 사용자 및 권한 외 접근 시 보호 로직

---

## 👨‍🏫 2. 강의 개설 (Course Registration) - 강사 전용

- [ ] **권한 기반 UI/UX 제어 (RBAC)**
  - [ ] **버튼 제어**: 강의 목록에서 수강생에게는 '강의 개설' 버튼 비노출
  - [ ] **접근 차단**: 수강생 권한으로 등록 페이지 진입 시 차단 처리
- [ ] **입력 필드 및 가공**
  - [ ] 강의명, 최대 수강 인원, 가격 필드 구현
  - [ ] 가격 입력 시 숫자 포맷팅 (천 단위 콤마 등 적용)
- [ ] **등록 프로세스**
  - [ ] '강의 개설' 버튼 클릭 시 진입 -> 정보 입력 -> 등록하기
  - [ ] 등록 완료 후 **강의 목록 페이지로 리다이렉트**

---

## 📚 3. 수강 신청 (Course Enrollment)

- [ ] **강의 목록 조회 및 정렬**
  - [ ] **무한 스크롤(Infinite Scroll)**
  - [ ] **아이템 정보**: 강의명, 강사명, 인원 현황(8/10), 포맷팅된 가격(200,000원)
  - [ ] **정렬 로직 상세 구현**:
    - [ ] 최근 등록순 (최신순)
    - [ ] 신청자 많은 순 (절대 인원수 기준)
    - [ ] 신청률 높은 순 (`신청자 / 정원` 비율 기준)
- [ ] **다중 신청 기능 (Multi-Selection)**
  - [ ] 아이템별 선택 체크박스 및 하단 고정 '수강 신청하기' 버튼
  - [ ] **예외 피드백**: 이미 신청한 강의 또는 마감된 강의 신청 시 알림 처리
  - [ ] 일괄 신청 완료 후 결과 확인 (성공 메시지 등)
- [ ] 강사 회원도 강의 수강 신청 가능

---

## 💎 품질 및 사용성 (Quality & UX)

- [ ] 모바일 웹 기준의 레이아웃 및 사용성 확보
- [ ] 로딩 상태 및 에러 핸들링을 통한 안정적인 UX 제공
- [ ] 효율적이고 버그 없는 코드 품질 유지
- [ ] 확장 가능한 애플리케이션 구조 설계
- [ ] 코드 및 문서 가독성 확보
