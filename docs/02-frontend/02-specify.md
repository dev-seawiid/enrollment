# Feature Specification: 수강신청 서비스 Frontend

**Date**: 2026-03-14 | **Status**: Implemented | **Input**: `docs/00-product/01-requirements.md`

---

## User Scenarios

### US-1 [P1] 회원가입 — 신규 사용자가 계정을 생성한다

수강생 또는 강사 역할을 선택하여 가입하고, 완료 후 자동 로그인되어 강의 목록으로 이동한다.

**Acceptance Scenarios**:

| #   | Given                   | When                                          | Then                                            |
| --- | ----------------------- | --------------------------------------------- | ----------------------------------------------- |
| 1   | 가입 페이지 진입        | 모든 필드를 유효하게 입력하고 "가입하기" 클릭 | 계정 생성 → 자동 로그인 → `/courses` 리다이렉트 |
| 2   | 비밀번호 입력 중        | 6자 미만 또는 영문/숫자 조합 미충족           | 실시간 유효성 에러 메시지 표시                  |
| 3   | 전화번호 입력           | `01012345678` 입력                            | `010-1234-5678`로 자동 포맷팅                   |
| 4   | 이미 가입된 이메일 입력 | "가입하기" 클릭                               | "이미 사용 중인 이메일입니다" 에러 표시         |
| 5   | 비밀번호 확인 불일치    | 비밀번호 ≠ 비밀번호 확인                      | "비밀번호가 일치하지 않습니다" 에러 표시        |

---

### US-2 [P1] 로그인 — 기존 사용자가 인증한다

이메일과 비밀번호로 로그인하여 세션을 획득한다.

**Acceptance Scenarios**:

| #   | Given                | When                                   | Then                                           |
| --- | -------------------- | -------------------------------------- | ---------------------------------------------- |
| 1   | 로그인 페이지 진입   | 유효한 자격 증명 입력 후 "로그인" 클릭 | 인증 성공 → `/courses` 리다이렉트              |
| 2   | 잘못된 비밀번호 입력 | "로그인" 클릭                          | "비밀번호가 일치하지 않습니다" 토스트 표시     |
| 3   | 이메일 입력 중       | `@` 이후 타이핑                        | 도메인 자동 완성 팝오버 표시 (gmail, naver 등) |
| 4   | 이미 로그인 상태     | `/login` 접근                          | `/courses`로 리다이렉트                        |

---

### US-3 [P1] 강의 목록 조회 — 사용자가 강의를 탐색한다

무한 스크롤로 강의 목록을 탐색하고, 정렬 기준을 변경한다.

**Acceptance Scenarios**:

| #   | Given          | When                  | Then                                |
| --- | -------------- | --------------------- | ----------------------------------- |
| 1   | 강의 목록 진입 | 페이지 로드           | 최근 등록순으로 첫 10개 강의 표시   |
| 2   | 목록 하단 도달 | 스크롤 다운           | 다음 페이지 자동 로드 (무한 스크롤) |
| 3   | 정렬 탭 클릭   | "신청자 많은 순" 선택 | 목록이 해당 기준으로 재정렬         |
| 4   | 로딩 중        | 다음 페이지 로딩      | 스켈레톤 카드 표시                  |
| 5   | 비로그인 상태  | `/courses` 접근       | `/login`으로 리다이렉트             |

---

### US-4 [P1] 수강 신청 — 사용자가 강의를 일괄 신청한다

여러 강의를 선택하여 한 번에 수강 신청하고, 결과를 확인한다.

**Acceptance Scenarios**:

| #   | Given            | When                  | Then                                     |
| --- | ---------------- | --------------------- | ---------------------------------------- |
| 1   | 강의 카드 클릭   | 체크박스 토글         | 선택 상태 변경 + 하단 배치 신청 바 표시  |
| 2   | N개 강의 선택    | "수강 신청하기" 클릭  | 배치 API 호출 → 결과 다이얼로그 표시     |
| 3   | 부분 성공        | 일부 강의만 신청 성공 | 성공/실패 항목을 구분하여 표시           |
| 4   | 마감된 강의      | 카드 클릭 시도        | 선택 불가 + "마감" 뱃지 표시             |
| 5   | 이미 신청한 강의 | 배치 신청에 포함      | 실패 사유: "이미 수강 신청한 강의입니다" |

---

### US-5 [P2] 강의 개설 — 강사가 새 강의를 등록한다

강사 역할의 사용자만 강의를 개설할 수 있다.

**Acceptance Scenarios**:

| #   | Given                  | When                           | Then                                           |
| --- | ---------------------- | ------------------------------ | ---------------------------------------------- |
| 1   | 강사로 로그인          | 강의 목록에서 "강의 개설" 클릭 | `/courses/new` 진입                            |
| 2   | 강의 정보 입력         | "등록하기" 클릭                | 확인 다이얼로그 → 등록 → `/courses` 리다이렉트 |
| 3   | 가격 입력              | `200000` 입력                  | `200,000` 포맷팅 표시                          |
| 4   | 수강생으로 로그인      | 강의 목록 진입                 | "강의 개설" 버튼 미노출                        |
| 5   | 수강생이 URL 직접 접근 | `/courses/new` 진입 시도       | `/courses`로 리다이렉트                        |

## Functional Requirements

| ID     | Requirement                                                  | Story |
| ------ | ------------------------------------------------------------ | ----- |
| FR-001 | 이메일, 비밀번호, 이름, 전화번호, 역할로 회원가입            | US-1  |
| FR-002 | 비밀번호: 6~10자, 영문 소/대문자 + 숫자 중 2종 이상 조합     | US-1  |
| FR-003 | 전화번호 자동 포맷팅 (IMask)                                 | US-1  |
| FR-004 | 이메일 + 비밀번호 로그인, JWT 세션 유지                      | US-2  |
| FR-005 | 비로그인 사용자 라우트 보호 → `/login` 리다이렉트            | US-3  |
| FR-006 | 강의 목록 무한 스크롤 (Intersection Observer, 페이지당 10개) | US-3  |
| FR-007 | 정렬: 최근 등록순 / 신청자 많은 순 / 신청률 높은 순          | US-3  |
| FR-008 | 다중 선택 + 배치 수강 신청 (부분 성공 지원)                  | US-4  |
| FR-009 | 마감 강의 선택 차단 + "마감" 뱃지 표시                       | US-4  |
| FR-010 | 강사 전용 강의 개설 (RBAC: 버튼 노출 + 라우트 보호)          | US-5  |
| FR-011 | 가격/수강인원 숫자 포맷팅 (천 단위 콤마)                     | US-5  |

## Key Entities

| Entity | Fields                                                                    |
| ------ | ------------------------------------------------------------------------- |
| User   | id, email, name, phone, role (STUDENT \| INSTRUCTOR)                      |
| Course | id, title, instructorName, maxStudents, currentStudents, price, createdAt |
| Enroll | enrollmentId, courseId, courseTitle, userId, enrolledAt                   |

---

## Success Criteria

- [ ] 회원가입 → 자동 로그인 → 강의 목록 도달까지 3단계 이내
- [ ] 무한 스크롤 시 1000개 카드에서 프레임 드랍 없음
- [ ] 배치 신청 결과를 성공/실패 구분하여 명확히 피드백
- [ ] 수강생 권한으로 강의 개설 경로 접근 시 100% 차단
- [ ] 모든 폼 유효성 검증이 실시간으로 동작
